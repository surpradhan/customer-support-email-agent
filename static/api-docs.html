POST /email/process

Process an incoming customer support email through the LangGraph agent workflow. Classifies the email, searches the knowledge base, drafts a response, and performs quality checks.

Request Body
from_email
string (email)
Customer's email address
subject
string
Email subject line
body
string
Email message body
conversation_id
string (optional)
Unique identifier for tracking this conversation thread
Example Request
curl -X POST http://localhost:8000/api/v1/email/process \ -H "Content-Type: application/json" \ -d '{ "from_email": "customer@example.com", "subject": "I need help with my order", "body": "I ordered item ABC123 but received the wrong product.", "conversation_id": "conv_12345" }'
Response
category
string
Classified email category (e.g., "complaint", "billing", "technical_issue")
sentiment
string
Detected sentiment (e.g., "positive", "neutral", "negative")
reply_body
string
AI-generated response to the customer
status
string
Status of email processing (e.g., "sent", "pending", "requires_followup")
requires_followup
boolean
Whether this email requires manual follow-up
conversation_id
string
Conversation identifier for tracking
Example Response
{ "reply_body": "Thank you for contacting us. We apologize for the inconvenience...", "category": "complaint", "sentiment": "negative", "status": "sent", "requires_followup": true, "conversation_id": "conv_12345" }
200 OK — Email processed successfully
GET /inbox/stats

Retrieve statistics about all emails in the inbox, including totals for processed, pending, and follow-up emails.

Response
total
integer
Total number of emails processed
sent
integer
Number of emails with responses sent
pending
integer
Number of emails still pending processing
requires_followup
integer
Number of emails requiring follow-up action
Example Request
curl http://localhost:8000/api/v1/inbox/stats
Example Response
{ "total": 42, "sent": 35, "pending": 5, "requires_followup": 2 }
200 OK — Stats retrieved successfully
GET /inbox

Retrieve a paginated list of all emails in the inbox with their analysis results.

Query Parameters
limit
integer (optional, default: 20)
Maximum number of emails to return (1-100)
offset
integer (optional, default: 0)
Number of emails to skip for pagination
Response
emails
array
Array of email objects with full details
total
integer
Total number of emails in the inbox
limit
integer
Limit applied to this request
offset
integer
Offset applied to this request
Example Request
curl "http://localhost:8000/api/v1/inbox?limit=10&offset=0"
Email Object Fields
email_id
string
Unique identifier for the email
from_email
string
Customer's email address
subject
string
Email subject line
body
string
Email message body
category
string
Classified category of the email
sentiment
string
Sentiment analysis result
reply_body
string
AI-generated response
status
string
Current status (e.g., "sent", "pending")
requires_followup
boolean
Whether follow-up is needed
created_at
string (ISO datetime)
Timestamp when email was received
200 OK — Emails retrieved successfully
GET /inbox/{email_id}

Retrieve complete details for a specific email, including the original message and AI analysis.

Path Parameters
email_id
string
The unique identifier of the email (required)
Example Request
curl http://localhost:8000/api/v1/inbox/email_abc123def456
200 OK — Email details retrieved
404 Not Found — Email with specified ID does not exist
GET /followups

Retrieve a list of all pending follow-ups across conversations.

Response
followups
array
Array of follow-up objects
Example Request
curl http://localhost:8000/api/v1/followups
Follow-up Object Fields
conversation_id
string
Conversation identifier
reason
string
Reason for follow-up
email_id
string
Associated email identifier
created_at
string (ISO datetime)
When the follow-up was created
200 OK — Follow-ups retrieved successfully
GET /followups/{conversation_id}

Get detailed information about a specific pending follow-up.

Path Parameters
conversation_id
string
The conversation identifier (required)
Example Request
curl http://localhost:8000/api/v1/followups/conv_12345
200 OK — Follow-up details retrieved
404 Not Found — Follow-up not found for this conversation
GET /health

Check if the API is running and responsive. Returns the current environment.

Example Request
curl http://localhost:8000/health
Example Response
{ "status": "ok", "environment": "development" }
200 OK — API is healthy and running
