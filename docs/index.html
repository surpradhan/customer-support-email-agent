Customer Support Email Agent

An AI-powered email triage and response system built with LangGraph and local LLMs via Ollama.

Python 3.12 LangGraph FastAPI Ollama FAISS Runs locally

How It Works

Incoming emails flow through a LangGraph state machine. Each stage is an async node that can read and update shared state. A quality-check loop ensures the draft meets standards before sending.

Pipeline Stages

1
Classify
The LLM categorizes the email into one of: complaint, inquiry, refund request, feedback, technical issue, or other. Sentiment (positive, neutral, negative, urgent) is detected at the same time.
2
Research
A semantic search query is generated and run against a FAISS vector index built from support documentation and FAQs. The top-3 matching passages are retrieved as context.
3
Draft
A response is generated using category-specific prompt templates that incorporate the customer's email, detected sentiment, and the retrieved knowledge base context.
4
Quality Check
A second LLM pass evaluates the draft for tone, accuracy, and completeness. If it doesn't pass, the draft is sent back for revision (up to 2 times). The check also flags whether a human follow-up is needed.
5
Send
The final response is recorded. If follow-up is required, a follow-up tracker entry is created. In a real integration, this is where the email would be dispatched.

Quick Start

Install dependencies python3.12 -m venv .venv && source .venv/bin/activate pip install -r requirements.txt cp .env.example .env # 3. Build the knowledge base index python -m src.knowledge_base.build_index # 4. Start the server uvicorn src.main:app --reload
